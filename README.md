# AiAdvent. Задание №1

## Задание

Отправьте один и тот же запрос, но:

- добавьте явное описание формата ответа;
- добавьте ограничение на длину ответа;
- добавьте условие завершения ответа (stop sequence или явную инструкцию).

Сравните ответы:

- без ограничений;
- с ограничениями;

Результат:
Один и тот же запрос с разным уровнем контроля ответа через API.

Формат:
Видео + Код

## Решение

[Ссылка на видео](https://disk.yandex.ru/i/3zhaRBXwiAfk2Q)

#### Параметры запроса

```json

  "temperature": "0.5"
```

#### Ответ модели

```
# Архитектурные стили для проектирования ИС

## 1. **Монолитная архитектура**
- Единое приложение с общей кодовой базой
- Все компоненты тесно связаны
- Простая разработка, сложное масштабирование

## 2. **Микросервисная архитектура**
- Система из независимых сервисов
- Каждый сервис отвечает за одну функцию
- Высокая масштабируемость и гибкость

## 3. **Слоистая (N-tier) архитектура**
- Разделение на слои: презентация, бизнес-логика, данные
- Классический подход, хорошая организация кода
- Слабая масштабируемость

## 4. **Событийно-ориентированная архитектура**
- Компоненты взаимодействуют через события
- Асинхронное взаимодействие
- Высокая гибкость и реактивность

## 5. **Архитектура на основе сервисов (SOA)**
- Переиспользуемые сервисы
- Слабая связанность компонентов
- Интеграция через ESB (Enterprise Service Bus)

## 6. **Serverless архитектура**
- Функции как единица развертывания
- Управление инфраструктурой провайдером
- Оплата по использованию

## 7. **Модульная архитектура**
- Система из независимых модулей
- Четкие интерфейсы между модулями
- Переиспользуемость компонентов

## 8. **Архитектура "Клиент-Сервер"**
- Разделение на клиентскую и серверную части
- Централизованная обработка на сервере
- Классический подход

## 9. **Архитектура CQRS**
- Разделение команд (записи) и запросов (чтения)
- Разные модели для чтения и записи
- Оптимизация производительности

## 10. **Гексагональная архитектура (Ports & Adapters)**
- Ядро приложения изолировано от внешних зависимостей
- Легко тестировать и менять реализацию
- Высокая модульность

## 11. **Архитектура на основе плагинов**
- Ядро + расширяемые плагины
- Динамическая загрузка функциональности
- Гибкость и модульность

## 12. **Облачная архитектура**
- Использование облачных сервисов
- Масштабируемость и отказоустойчивость
- Управление ресурсами провайдером

Выбор стиля зависит от требований проекта, масштаба, команды и целей развития системы.
```
